Address review feedback on settings-autosave PR

- Evict idle useAutosave queue entries so the module-level Map doesn't
  grow with every distinct queryKey used in a session.
- Switch queue keying to React Query's `hashKey` for consistency with
  cache-key semantics and robustness against function/regex values.
- Lock the coalesce fn to the first dispatcher on a shared queryKey so
  cross-component merges are deterministic; dev-warn on divergent fns.
- Dev-warn from `applyOrgSettingsPatch` when a partial nested object
  would shallow-overwrite existing keys (agent_config, product_context).
- Reject non-object settings patches in `mergeSettingsJSON` rather than
  silently no-opping on `null`/scalar/array payloads.
- Add tests for shared-queryKey status sync, numeric-field debounce
  note, and the new merge rejection cases.
