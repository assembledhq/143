# Contributing

Thanks for contributing to 143.

## Before You Start

- Read the architecture overview in `docs/design/overall.md`
- Check existing issues or open a discussion before making large changes
- Follow the repo instructions in `AGENTS.md` and any directory-specific `AGENTS.md`

## Development Expectations

- Add tests before implementation changes
- Run the relevant checks before submitting changes
- Keep changes focused and document any notable design updates under `docs/design/`

## Contribution License

Unless explicitly stated otherwise, any contribution intentionally submitted for
inclusion in this repository is licensed under the same MIT License that covers
the project.

Our contribution model is inbound=outbound: by submitting a contribution, you
represent that you have the right to submit it under the repository's MIT
License, without additional terms or conditions.

## Security Issues

Do not report security vulnerabilities in public issues. Follow `SECURITY.md`
instead.
